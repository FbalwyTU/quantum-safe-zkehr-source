# Packaging Audit

Repository prepared: `quantum-aware-zkehr-source`

Manuscript:

> Quantum-Aware Extension of Blockchain-Based Zero-Knowledge EHR Systems: IBM Quantum Feasibility and ML-KEM Key-Release Migration

This audit records the source-project inspection performed before packaging the public reproducibility artifact. Only files inside `quantum-aware-zkehr-source/` are modified by the packaging process.

## 1. Candidate Source Directories Found

- `contracts/`: Solidity verifier and baseline contracts.
- `circuits/`: Circom access circuit and example input.
- `lib/`: JavaScript helpers for record encryption, IPFS interaction, proof formatting, verifier access, and secure-record workflow.
- `scripts/`: baseline deployment and record-retrieval scripts.
- `test/`: Hardhat contract and workflow tests.
- `evaluation/`: baseline experiment harness scripts.
- `ignition/`: Hardhat Ignition module.
- `implementation_tools/phase1`, `phase2`, `phase2_5`: baseline reproduction and validation scripts.
- `quantum_extension/qiskit/`: Qiskit/Aer simulator, QRNG, toy access-policy, IBM readiness, and IBM hardware execution scripts.
- `quantum_extension/pqc/`: ML-KEM/Open Quantum Safe utilities, smoke tests, and RSA-OAEP vs. ML-KEM comparison harness.

## 5. Files and Directories Explicitly Excluded

- `quantum-aware-zkehr-source/.git/`: preserved as the cloned repository metadata, not copied from the parent.
- `node_modules/`: dependency cache; regenerated by `npm install` or `npm ci`.
- `.venv/`, `venv/`, `quantum_extension/qiskit/.venv/`, `quantum_extension/pqc/.venv/`: local Python environments; regenerated from requirements files.
- `__pycache__/`, `.pytest_cache/`, cache folders, `cache/`, `artifacts/`: generated build/runtime artifacts.
- `baseline_results/phase2_5/ipfs_repo/`: generated local IPFS repository and LevelDB data.
- `baseline_results/*/runtime/`, `baseline_results/*/requester-demo/`: generated runtime data and requester key material.
- `baseline_results/phase2_5/zk_artifacts/`: generated Circom/snarkjs artifacts, witness files, ptau/zkey files, and generated proof files; regenerated by the documented commands.
- `downloads/`, `Encrypted_EHR.pdf`, and other PDF record fixtures: excluded to avoid distributing record-like PDFs or binary input data.
- `go-ipfs_v0.18.1_darwin-amd64.tar.gz` and `go-ipfs/`: platform-specific binary installer assets; reviewers should install Kubo/IPFS externally.
- `paper_assets/`, `figures/`, `overleaf/`: manuscript and paper assets are excluded.
- `implementation_reports/final_research_integration_master_handoff_for_claude.md`, `baseline_code_architecture_analysis.md`, and other handoff/planning-style Markdown: excluded as drafting/planning material.
- `project_snapshots/`: generated snapshots and archives containing duplicated source, runtime state, and excluded artifacts.
- Command logs such as `*_command_log.txt`: local logs, often containing machine-specific paths.
- `.DS_Store` and OS metadata files.

## 6. Missing Reproducibility Components

- No project-level Python requirements files were present; packaging creates clean `requirements-qiskit.txt` and `requirements-pqc.txt` from observed project versions.
- No production or manuscript-level statistical aggregation script was found beyond the existing CSV/JSON result writers.
- IBM Quantum hardware reruns require a reviewer-owned IBM Quantum account, token, visible backend, queue availability, and calibration state; no token is packaged.
- IPFS end-to-end workflow scripts require a local Kubo/IPFS daemon and a non-sensitive source record supplied by the user or the included synthetic sample.
- Full Circom/snarkjs proving artifacts are not packaged; they are regenerated by the documented Groth16 workflow.
